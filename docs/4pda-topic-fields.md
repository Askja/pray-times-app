## Описание программы для большого поля

Вставлять в поле `Описание программы` внизу формы. Это BBCode, форум его нормально съест.

```bbcode
[b]Prayer Times[/b] - Android-приложение для расчета времени намаза, направления киблы, исламского календаря и напоминаний.

[b]Основные возможности:[/b]
[list]
[*]Расчет Fajr, Sunrise, Dhuhr, Asr, Maghrib и Isha.
[*]Автоматический подбор метода расчета по локации.
[*]Ручной выбор метода расчета и мазхаба для Asr.
[*]Кибла с компасом и углом направления.
[*]Текущая геолокация, поиск города/адреса и сохраненные локации.
[*]Встроенная база городов и fallback-геокодинг через внешние сервисы.
[*]Хиджри-календарь и основные исламские праздники.
[*]Режим Рамадана: сухур, ифтар и таравих.
[*]Сунна-намазы, нафль-намазы и рекомендуемые посты.
[*]Уведомления до начала, в начале и перед окончанием намаза.
[*]Азан при начале намаза, выбор звука, вибрация и профили уведомлений по каждому намазу.
[*]Икама/джамаат offsets и заметки для сохраненных локаций.
[*]Экспорт месячного расписания в PDF и ICS.
[*]Виджет с текущим и следующим намазом, прогрессом, countdown и киблой.
[*]Экспорт и импорт настроек/локаций в JSON.
[*]Светлая и темная темы.
[*]Поддержка RTL и многоязычных ресурсов.
[/list]

[b]Разрешения:[/b]
[list]
[*]Геолокация - для определения текущего города и расчета времени намаза.
[*]Интернет - для поиска адресов, карт и списка мечетей.
[*]Уведомления, точные будильники, запуск после перезагрузки и вибрация - для надежных напоминаний о намазе.
[/list]

[b]Важно:[/b]
Праздники считаются по табличному календарю Хиджры. Для реального объявления праздников все равно решает локальное наблюдение луны.
```

## Поля формы по порядку

### Название программы

Указать без версии:

```text
Prayer Times
```

### Описание программы

Короткая строка без точки в конце, если хочется в стиле 4PDA:

```text
Время намаза, кибла, хиджри-календарь, исламские праздники и напоминания
```

### Краткое описание возможностей

Одним предложением:

```text
Расчет времени намаза, направление киблы, уведомления, азан, календарь Хиджры, сунна-намазы и посты
```

### Версия

Берется из `app/build.gradle.kts`:

```text
0.1.1
```

### Требуемая версия Android

Берется из `minSdk = 26`:

```text
Android 8.0+ (API 26+)
```

В выпадающем списке выбирай `8.0 (API Level 26)`, если такая опция есть.

### Иконка приложения

Загрузить PNG/JPEG/WEBP. Требования формы:

- размер файла до 192 МБ;
- картинка не меньше 120x120;
- картинка не больше 200x200.

Готовый файл:

```text
docs/4pda/prayer-times-icon-4pda.png
```

### Скриншоты

Загрузить реальные скриншоты, без промо-картинок. Актуальный набор:

```text
screenshots/pray-times-main-current.png
screenshots/pray-times-location-top-current.png
screenshots/pray-times-location-note-current.png
screenshots/pray-times-settings-current.png
screenshots/pray-times-settings-iqama-current.png
screenshots/pray-times-settings-backup-actions-current.png
```

### Русский интерфейс

```text
Да
```

В проекте есть `values-ru`, значит поле не врем, все честно.

### Особенности

Отметить:

```text
[widg]
```

Не отмечать, если это не планируется отдельно:

```text
[OEM]
[HD]
[ATV]
```

### Необходимы права root?

```text
Нет
```

### Разработчик

```text
Askja
```

Если хочешь бренд вместо ника:

```text
Admiral
```

### Домашняя страница / источник

Если Google Play ещё нет, по правилу 2.4 нужен источник с авторством и именем пакета. Лучший вариант:

```text
Публичный Git-репозиторий с исходным кодом приложения
```

В поле ссылки указать URL репозитория:

```text
https://github.com/Askja/pray-times-app
```

### Google Play / RuStore / AppGallery / F-Droid

Оставить пустым до публикации в магазинах. После принятия в F-Droid вставить прямую ссылку на карточку:

```text
https://f-droid.org/packages/ru.admiral.praytimes/
```

До принятия в F-Droid не писать, что приложение там уже есть. Это не маркетинг, это самострел.

### Имя пакета

```text
ru.admiral.praytimes
```

### YouTube / RUTUBE

Оставить пустым, если нет видеообзора или демо.

### Файлы

Загрузить подписанный release APK:

```text
app/build/outputs/apk/release/pray-times-0.1.1-4pda-signed.apk
```

Не загружать:

```text
app-release-unsigned.apk
app-release-4pda-aligned.apk
pray-times-0.1.1-4pda-aligned.apk
```

Aligned APK - это ещё не подписанный APK. Samsung/One UI такой файл закономерно пошлёт, и будет прав, зараза.

### Ссылка на APK

Использовать только если APK доступен с сайта разработчика. Если файл загружен прямо в тему, поле можно оставить пустым.

### Источник, авторство и пакет для модератора

Если модератор просит подтверждение по правилу 2.4, вставить в комментарий или шапку:

```text
Источник: https://github.com/Askja/pray-times-app
Разработчик: Askja, автор темы
Имя пакета: ru.admiral.praytimes
```

После публикации F-Droid можно использовать:

```text
Источник: https://f-droid.org/packages/ru.admiral.praytimes/
Исходный код: <ссылка на публичный Git-репозиторий>
Разработчик: Askja, автор темы
Имя пакета: ru.admiral.praytimes
```

## Что проверить перед публикацией

- Версия в теме совпадает с `versionName`.
- APK подписан релизным ключом и проходит `apksigner verify`.
- В теме указан пакет `ru.admiral.praytimes`.
- Скриншоты актуальные, полноразмерные, реальные и на русском интерфейсе.
- Иконка взята из `docs/4pda/prayer-times-icon-4pda.png`, а не из случайного скриншота.
- В описании нет обещаний про магазин, если приложения там еще нет.
- Для правила 2.4 указан источник: публичный Git/F-Droid, автор `Askja`, пакет `ru.admiral.praytimes`.
- Если используешь геокодеры с API-ключами, в публичный APK не попали тестовые/личные ключи.
